# スキルマッチングプラットフォーム MVP要件

## 目的
- AI-chanが不在でも、メンバーのスキル情報が公開され、問い合わせと決済が継続できる状態を作る
- まずは手作業の運営を許容しつつ、承認・通知・決済の自動化だけ先に実装する

## スコープ
- メンバー登録（下書き）
- 運営承認（Supabase管理画面でチェック）
- 検索付きメンバー一覧公開
- 直接問い合わせ送信（メンバー + 運営へメール）
- Stripe Checkoutによる単発決済
- Stripe Webhookによる入金記録

## 非スコープ（Phase 2以降）
- 複雑な推薦アルゴリズム
- 取引チャット機能
- 契約書自動生成
- 返金自動化

## 機能要件
1. メンバー登録
  - 必須: 表示名、スキルタグ、連絡先メール
  - 登録時は `isApproved = false`
2. メンバー検索
  - キーワード（肩書き、スキルタグ）
  - 料金レンジ（minRate / maxRate）
  - 公開対象は `isApproved = true` のみ
3. 問い合わせ
  - 依頼主情報と相談内容を保存
  - メンバーと運営に同報メール
4. 決済
  - サービス名と金額を指定してCheckout Session生成
  - 完了時にOrderレコードを生成

## 非機能要件（最小）
- 稼働率: 95%以上（MVP）
- 応答目標: 3秒以内（一覧API）
- セキュリティ: 秘密鍵は環境変数管理
- 監査性: 問い合わせ・注文はSupabaseに記録
